What We're Solving

Drug–drug interactions (DDIs) occur when two or more medications are taken together and one affects the pharmacokinetics, pharmacodynamics, or safety of another. These interactions can cause adverse events, reduced efficacy, or toxicity—especially when combining novel drugs or when clinical trial data is limited.

Pharmagen addresses this by predicting interaction risk for novel drug combinations using chemical structure (SMILES) and structural similarity. Instead of relying solely on curated databases (which often lack newer drugs), we use computational methods to estimate risk based on molecular overlap and shared metabolic pathways.

Core Problem

  • Clinical DDI databases (e.g. NCATS, DrugBank) cover known pairs but miss novel combinations.
  • Polypharmacy is common—patients often take 5+ medications—increasing interaction risk.
  • Vaccines and biologics add complexity; LNP components and immune modulation can interact with small molecules.
  • Pharmacogenomics (CYP450, etc.) affects metabolism; genotype influences who is at risk.

Our Approach

We combine deterministic logic (PubChem lookup, Tanimoto similarity) with AI-powered explanation (OpenAI) and clear extension points for trained ML models. The API provides:

  • Drug search & lookup — Resolve drug names to SMILES via PubChem
  • Interaction prediction — Structural similarity (Tanimoto) + confidence scoring
  • Timeline visualization — 72-hour risk and concentration curves
  • AI explanation — Mechanistic insights (CYP450, metabolic competition)
  • Q&A — Ask questions about interactions with context

Setup Guide

Prerequisites

  • Python 3.10+ — For the API
  • Node.js 18+ — For the frontend (optional)
  • Git

1. Clone the Repository

git clone https://github.com/Topupchips/HackIllinoisWinningIdea.git
cd HackIllinoisWinningIdea

2. Install Backend Dependencies

pip install -r backend/requirements.txt

3. Install Frontend Dependencies (Optional)

npm install

4. Environment Variables (Optional)

cp .env.example .env

Edit .env for VITE_API_URL, OPENAI_API_KEY, API_KEY, etc.

Run Guide

Run the API

python backend/run_local.py

API runs at http://127.0.0.1:8000.

Run the Frontend

npm run dev

App runs at http://localhost:5173.

Quick Verification

curl http://127.0.0.1:8000/v1/health
curl "http://127.0.0.1:8000/v1/drugs/search?q=aspirin"

Optional: AI Features

export OPENAI_API_KEY=sk-your-key
python backend/run_local.py

Optional: API Key Auth

export API_KEY=your-secret-key
python backend/run_local.py

Then include X-API-Key header in requests.
